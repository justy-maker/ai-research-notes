## 摘要

介紹如何結合 Nano Banana（圖像生成）+ Gemini 3（語言模型）來實現 S 級 UI 設計流程。

### 核心結論

圖像生成模型 + 語言模型的組合，遠比單純用 coding model 做 UI 設計強得多。

### 4 步流程

#### 1. 設計規格（Design Spec）
- 先做規劃，而不是直接讓模型「隨便設計」
- 把 UI 需求拆解成更小的設計任務
- 一次只讓模型專注一個點（布局/信息層級/風格/組件）
- 先參考成熟產品或設計網站，明確方向再開始生成

#### 2. Nano Banana 生成 UI
- 圖像生成模型在 UI 創意層面遠勝於純代碼模型
- 可設計出更大膽的布局
- 更有風格感的組件
- 更接近真實產品的視覺方案
- 創意密度提升 10 倍

#### 3. 資產提取（Assets Extraction）
- 從生成的 UI 中提取複雜的圖形資產
- 處理用代碼很難快速實現的視覺元素
- 配合視頻生成模型做動畫效果
- 極大降低「設計 ↔ 實現」之間的摩擦成本

#### 4. 案例實戰（Case Study）
- Plan Mode + 圖像生成能力整合進 SuperDesign.dev
- 可一步步生成設計
- 導入自己的品牌資產
- 把整個 UI 設計流程直接串起來

## 關鍵要點

- 圖像生成模型 + 語言模型 = 強大組合
- 分步驟設計比一次性設計效果更好
- Nano Banana 特別適合 UI 創意生成
