## 操作步驟

### 步驟 1：建立 v0 帳號並開始
1. 前往 v0 網站，建立帳號並登入
2. 在 Dashboard 可以看到過去的對話記錄
3. 點擊開始新的 Chat

### 步驟 2：輸入 Prompt 生成 UI
1. 輸入需求描述，例如：「生成一個多步驟精靈表單，用於 SaaS 產品的用戶 onboarding」
2. 按 Enter 送出
3. 左側是 AI 對話，右側是程式碼和預覽

### 步驟 3：理解 v0 的預覽環境
- **重要觀念**：v0 的預覽不是本地環境（localhost）
- 需要網路連線和連接 v0 伺服器才能預覽
- 本地環境允許離線開發和測試

### 步驟 4：管理聊天記錄
- **追問功能**：可以在現有基礎上繼續修改
- **收藏功能**：收藏重要的對話
- **重新命名**：給對話一個有意義的名稱
- **查看程式碼**：點擊右側查看完整程式碼

### 步驟 5：理解 UI 與後端的區別
- v0 生成的是**純前端 UI**
- 沒有連接後端，表單資料不會儲存
- 適合用於：
  - 沙盒測試
  - UI 原型設計
  - MVP 快速開發

### 步驟 6：匯出到 Cursor AI
1. 點擊「Add to Codebase」按鈕
2. 複製提供的命令

### 步驟 7：在 Cursor AI 設置專案
1. 開啟 Cursor AI
2. 開啟終端機（Terminal）
3. 建立新資料夾：
   ```bash
   mkdir v0-good
   cd v0-good
   ```
4. 開啟該資料夾

### 步驟 8：匯入 v0 程式碼
1. 在終端機貼上從 v0 複製的命令
2. 出現安裝相依套件提示時，輸入 `Y` 確認
3. 回答設定問題：
   - 是否開始新 Node.js 專案？→ Y
   - App 名稱 → 自訂名稱（如 my-new-v0）
   - 樣式選擇 → New York
   - 基本顏色 → Neutral
   - 是否使用 CSS 變數做主題？→ Yes

### 步驟 9：部署到本地環境
1. 進入專案資料夾：
   ```bash
   cd my-new-v0
   ```
2. 啟動開發伺服器：
   ```bash
   npm run dev
   ```
3. 等待「ready」訊息出現
4. 點擊 localhost:3000 連結

### 步驟 10：即時修改與預覽
- 在 Cursor 中修改程式碼
- 儲存後（Cmd+S），瀏覽器會自動反映變化
- 這就是本地開發環境的優勢

## 重點功能

### v0 平台特色
1. **對話式 UI** - 用自然語言描述需求
2. **即時預覽** - 邊生成邊看效果
3. **Fork 功能** - 複製他人的作品繼續開發
4. **分享功能** - 公開分享你的設計
5. **匯出功能** - 一鍵匯出到 Cursor AI

### 適用場景
- 快速原型設計
- UI 沙盒測試
- 學習 React/Next.js 元件寫法
- MVP 開發

### 限制說明
- 預設只生成前端程式碼
- 沒有後端資料處理
- 需要另外開發 API 和資料庫連接

## 學習筆記

### 💡 關鍵觀念
1. **v0 = 前端 UI 生成器**，不是全端解決方案
2. **Localhost 環境**可以離線開發、即時反映變化
3. **Cursor AI** 是用來進一步開發和部署的工具

### ⚡ 快速指令
```bash
# 建立資料夾
mkdir project-name

# 進入資料夾
cd project-name

# 啟動開發伺服器
npm run dev
```
