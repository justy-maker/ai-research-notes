## 📌 核心觀點

Liam Ottley 認為 Claude Code 搭配的 AIOS（AI Operating System）是自 ChatGPT 以來最令人興奮的發展。他計畫在 30 天內用 Claude Code 自動化 60-70% 的業務任務，並認為這比 OpenClaw/Claudebot 更強大。

## 📝 AIOS 架構（層層疊加）

### 核心概念
- 商業模型（Business Model）是核心，AIOS 是圍繞它的自動化方法論
- 透過 Claude Code 的 harness（工具連接層）驅動
- 用 Telegram 作為行動端介面

### 模組層級
1. **Context OS** — 資料夾結構、文件、業務上下文
2. **Data OS** — 整合所有數據源（P&L、GA、YouTube、行銷、銷售）到本地資料庫，建構儀表板
3. **Meeting Intelligence OS** — 串接 Fireflies/Fathom API，可查詢過去會議內容
4. **Slack OS** — 拉取 24 小時內所有 Slack 訊息進行分析
5. **Daily Brief OS** — 整合會議、Slack、業務脈絡，產出每日簡報 + SWOT 分析 + 內容建議
6. **Productivity OS** — 全面的任務管理，透過 Telegram 對話更新
7. **Capture OS** — 串接 iMessage、WhatsApp、Gmail，早晨統一簡報處理
8. **Content OS** — 編輯、構思、腳本等內容製作流程

### Claude Code vs OpenClaw 的差異
- OpenClaw 是把模型從 Claude Code 的精心打造 harness 中拔出，放進較簡陋的 harness
- 例：Claude Code 原生支援網頁搜尋，OpenClaw 需要額外安裝
- OpenClaw 初學者友善但會撞到天花板，Claude Code 則無上限
