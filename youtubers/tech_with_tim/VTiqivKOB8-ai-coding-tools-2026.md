## 影片摘要

本影片介紹了講者在 2026 年每天使用的幾款 AI 編碼工具，包括 OpenClaw (一個本地運行的 AI 代理協調層)、Claude Code (一個輕量級的終端內編碼代理)、Cursor (一個基於 VS Code 的 AI 程式碼編輯器) 和 Warp (一個全功能的 AI 終端)。講者分享了這些工具的優缺點以及各自適用的場景，並強調了安全使用 AI 工具的重要性。

## 核心重點

*   **AI 編碼工具的快速發展：** 軟體開發正在快速變化，新的 AI 工具不斷湧現。
*   **OpenClaw：** 本地運行的 AI 代理協調層，功能強大但設置複雜，需注意安全性。
*   **Claude Code：** 輕量級的終端內編碼代理，擅長生成程式碼，但並非最佳選擇。
*   **Cursor：** 基於 VS Code 的 AI 程式碼編輯器，適合小修改、重構和需要仔細審查的專案。
*   **Warp：** 全功能的 AI 終端，擅長後端基礎設施和 DevOps 相關任務，提供命令自動完成等功能。
*   **安全第一：** 使用 AI 編碼工具時，務必注意資料安全，避免洩露個人資訊。

## 詳細內容

*   **OpenClaw：** 講者花費 10 小時設置 OpenClaw，並將其連接到各種工具和沙箱帳戶。它運行在虛擬私有伺服器上，可以通過 Telegram 進行消息傳遞和監控。講者強調，OpenClaw 的安全性非常重要，設置不當可能導致資料洩露。
*   **Claude Code：** 講者認為 Claude Code 是一個優秀的編碼代理，但並非像其他人所說的那樣是最佳選擇。他仍然會使用其他工具。
*   **Cursor：** 講者最常使用的 AI 程式碼編輯器，基於 VS Code，具有 AI 輔助功能。適合小修改、重構和需要仔細審查的專案。講者喜歡 Cursor 的原因是因為它與 VS Code 非常相似，使用起來很熟悉，並且 AI 代理不會妨礙他的工作流程。
*   **Warp：** 講者使用 Warp 處理後端基礎設施和 DevOps 相關任務。Warp 具有命令自動完成功能，可以簡化終端操作。講者喜歡在多個終端實例中同時使用 Warp。
*   **工具選擇：** 講者會根據不同的專案需求選擇不同的 AI 編碼工具。對於需要仔細審查的大型程式碼庫，他會選擇 Cursor。對於 DevOps 自動化任務，他會選擇 Warp。
