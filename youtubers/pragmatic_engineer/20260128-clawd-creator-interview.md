## 📝 核心摘要

Peter Steinberger，PSPDFKit（被超過 10 億設備使用的 PDF 框架）創辦人，在燃盡離開科技圈 3 年後，回歸並用 AI Agent 打造了爆紅的個人助理專案 **Clawd/Cloudbot**。這集訪談深入探討他如何用全新方式開發軟體——不再逐行審查程式碼，而是成為 AI 的「建築師」。

---

## 🎯 關鍵論點

### 1. 「我部署不讀的程式碼」不是瘋話
- **閉合迴圈原則**：讓 AI 能自行驗證、測試、除錯
- 程式碼可編譯、可 lint、可執行、可驗證輸出
- 這就是 AI 在程式設計上比創意寫作更強的原因

### 2. PR 應該叫「Prompt Request」
- Peter 更在意 **prompt 的品質** 而非程式碼本身
- 「我讀 prompt 比讀 code 更多，因為 prompt 告訴我你的思考過程」
- 傳統 Code Review 對他來說已經死了

### 3. 從 Vibe Coding 到 Agentic Engineering
- 他不喜歡「Vibe Coding」這詞，更喜歡稱為「Agentic Engineering」
- 「Vibe Coding 從凌晨 3 點開始」（暗示那是狀態不好時的亂搞）

---

## 🛠️ 他的工作流程

### 並行 Agent 管理
```
主任務（40 分鐘）→ 設計完成 → 開始執行
    ↓
切換到任務 B（20 分鐘）→ 同時運行
    ↓
切換到任務 C → 任務 A 完成 → 回去檢查
```

- 同時跑 **5-10 個 Agent**
- 像打星海爭霸：主基地 + 多個分礦
- 像西洋棋大師同時下多盤棋

### 工具選擇

| 工具 | 評價 |
|------|------|
| **Codex (OpenAI)** | 「被低估了」，幾乎每個 prompt 都給正確結果 |
| **Claude Code** | 品類定義者，但會跳過檔案、過度自信 |
| **MCP** | 「拐杖」—— CLI 更高效，可串連、可過濾 |

### 為什麼 CLI > MCP
- MCP 要預載所有函數說明（佔 context）
- 無法鏈式處理、無法用 jq 過濾
- CLI 讓 model 可以腳本化、組合命令

## 💡 對軟體工程的重新定義

### 傳統 vs AI 時代

| 面向 | 傳統 | AI 時代 |
|------|------|---------|
| PR | 可能等一週 | 幾分鐘內完成 |
| Code Review | 逐行審查 | 讀 prompt 更有價值 |
| CI | 等 10 分鐘 | 本地 Agent 跑完才推 |
| 重構 | 可能需 2 週 | Codex 3 小時搞定 |
| 文件 | 不想寫 | 讓 AI 寫，品質更好 |
| 測試 | 討厭寫 | 變成架構設計的一部分 |

### 「用 AI 反而讓你成為更好的工程師」
> 「你必須思考架構讓它更容易驗證，因為驗證就是讓東西變好的方式。」

## 🔮 對產業的預測

### 企業採用 AI 會很困難
- 需要重新定義公司運作方式
- 需要高 agency、高能力的人（非傳統分工）
- 可能只需要 **30% 的人力**

### 新手建議
1. **保持無限好奇**
2. 不需要寫很多程式碼，但需要建造東西獲取經驗
3. 有無限耐心的機器可以解答所有問題
4. **玩就對了**——像去健身房，一開始會痛苦但會進步

### 求職看什麼
- GitHub 活躍、做 Open Source
- 熱愛這個「遊戲」的人

## 🏠 Clawd/Cloudbot 專案

### 願景：超個人化助理
- 不是發「早安」郵件的那種
- 而是：
  - 「你 3 週沒聯絡 Thomas 了，他現在在城裡，要打個招呼嗎？」
  - 「每次你見完某人都很沮喪，為什麼？」
  - 類似電影《Her》的未來

### 技術亮點
- 可以打電話訂餐廳
- 可以控制智慧家電、看螢幕
- 可以自我更新（`update yourself`）
- Agent 編輯自己的設定檔
- 一週內從 100 星 → 3,300 星
- 已合併 500+ PR（他自稱「人肉 merge 按鈕」）

### 支援平台
- WhatsApp、Signal、iMessage、Discord、Slack、MS Teams、Matrix

## 🎙️ 金句摘錄

> 「那些程式碼很多都是無聊的管道工程。資料進來一種格式，解析成另一種，存進資料庫又是另一種，輸出成 HTML 又變了。我們就是在不同形狀間按摩資料。」

> 「我不再只管理一個員工，我管理 5-10 個，它們同時在不同東西上工作。」

> 「我現在從沒工作這麼拼，即使有公司的時候也沒有。不是因為必須，而是太上癮、太好玩了。」

> 「閉合迴圈——這就是讓 AI 工作的全部秘密。」

> 「我一天可以有 600 個 commit，而且不是垃圾。」

## 📌 個人筆記

這集訪談非常有啟發性，Peter 代表了一種新型態的開發者：

1. **Builder mindset**：比起「寫程式」，更像是「塑造產品」
2. **系統架構師 + AI 管理者**：不寫具體程式碼，但掌控全局
3. **極致的反饋迴圈設計**：這是讓 AI 有效的關鍵

值得注意的是，他 3 年沒碰電腦回來後，反而因為「沒有舊習慣包袱」而更快接受新工具。這對有經驗的開發者是個警訊——**有時候經驗反而是阻力**。
