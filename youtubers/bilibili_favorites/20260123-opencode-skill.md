## 📝 核心摘要

介紹 **OpenCode**（類似 Claude Code 的開源替代品）和 **Skills**（AI 增強插件）的使用與原理。Skills 讓模型獲得原本不支持的能力（如讀取 PDF）。

---

## 🎯 OpenCode 功能演示

### 本地文件操作
- 直接操作本地文件系統
- 可在終端打開、定位文件夾
- 修改文件名等操作

### 創作工作流示範
建立「自動型弱勢頭」內容創作系統：
1. 創建選題文件夾
2. 包含：選題、故事內容、分作說明
3. 生成 7 個選題

### 模式切換
- **Plan 模式**：不修改文件，用於規劃大型任務
- **Do 模式**：實際執行修改

## 🛠️ Skills 介紹

### 什麼是 Skills？
- AI 的增強插件
- 讓模型獲得原本不支持的能力
- 必須配合 AI 一起使用

### 官方 Skills 包
ASRAP 官方 Skills 包含：
- PDF 處理
- PPT 處理
- 表格處理
- 等等

## 📦 自己創建 Skills

### 基本結構
```
my-skill/
├── SKILL.md          # 說明文檔（必須）
├── scripts/          # 執行腳本（可選）
└── resources/        # 資源文件（可選）
```

### SKILL.md 必須包含
1. 名稱
2. 描述
3. **觸發條件**：什麼時候調用這個 Skill
4. 具體指令要求

### 注意事項
1. **全局 vs 項目**：可設為全局或僅當前項目可用
2. **路徑規範**：必須在 `.opencode/skills/` 目錄下
3. **不同平台**：Claude Code 用 `.claude/`，Cursor 用 `.cursor/`

## 💡 Skills vs MCP 的區別

| 對比 | Skills | MCP |
|------|--------|-----|
| 加載時機 | **按需加載**（觸發時才載入） | 每次完整加載所有說明 |
| 性能 | 更高效 | 佔用更多上下文 |
| 靈活性 | 條件觸發 | 始終可用 |

> Skills 核心特色：**按需加載**，只有符合條件時才觸發

## 📌 個人筆記

Skills 的按需加載機制很聰明：
- MCP 每次都要載入所有工具說明（浪費 context）
- Skills 只在需要時才載入具體指令

這對於複雜工作流非常有幫助，不會讓上下文被不相關的工具說明佔滿。
