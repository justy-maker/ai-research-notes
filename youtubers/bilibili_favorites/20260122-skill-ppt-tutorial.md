## 📝 核心摘要

通過實作「自動生成 PPT」的 Skill，帶大家理解 Skills 和 MCP 的區別。使用 Cursor 免費功能，**不用寫代碼**，用自然語言就能快速開發 Skill。

---

## 🎯 Skills 三層結構

```
SKILL.md
├── 元數據（Metadata）
│   └── AI 一開始就能看到
├── 指令（Instructions）  
│   └── 確認使用後才載入
└── 資源文件（Resources）
    └── 按需載入（知識庫、模板等）
```

### 載入流程
1. AI 上下文最初只有**元數據**
2. 確認使用 Skill 後，載入**指令**
3. 根據需要，載入相關**資源文件**
4. 綜合所有信息給出最終答案

---

## ⚔️ Skills vs MCP 對比

| 對比 | Skills | MCP |
|------|--------|-----|
| 重點 | **提示詞** | **功能調用** |
| 機制 | 按需加載提示詞 | 一次性全量加載 |
| Token 消耗 | 低 | 高 |
| 複雜度 | 可控 | 隨工具數增加 |

> **Skills 最大好處**：大幅降低 Token 消耗與提示詞複雜度

---

## 🛠️ 實作：自動 PPT Skill

### 創建方法
1. 用 `Crit` Skill 快速創建模板
2. 告訴 Cursor 你的需求（自然語言）
3. 自動生成兩個分工明確的 Skill

### 生成的 Skill 分工
| Skill | 職責 |
|-------|------|
| **思考 Skill** | 聯網搜索、讀取文件、提取核心觀點、創作 PPT 大綱 |
| **執行 Skill** | 將大綱通過代碼轉化為實際 PPT |

### 測試效果
輸入：「分析一下 2025 年新能源趨勢的發展，製作一份 PPT」
- 自動聯網搜索資料
- 生成結構清晰的大綱
- 包含數據、圖表的 PPT
