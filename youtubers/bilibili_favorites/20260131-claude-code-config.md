## 📝 核心摘要

介紹來自 **Anthropic 黑客松冠軍** 分享的 Claude Code 配置專案（GitHub 3.6 萬星），經過 10 個月密集使用與實戰驗證，將常用的 AI 技巧整理成規範化目錄。

---

## 🎯 核心組件

### 1. AI 代理（Sub-agents）
將不同任務分配給專門的代理：
- **Bug 修復代理**：專門處理 bug
- **大改修復代理**：處理大型重構
- **代碼審查代理**：專門做 code review

每個代理都有：
- 專屬名稱與描述
- 可使用的工具列表
- 指定的模型（如某些用 Gemini）

### 2. 四個五色（Workflows）
預製的工作流程，已在生產環境驗證：
- **Plan**：做計劃（作者最常用，比 Cursor 自帶的好用）
- 其他命令行工具

### 3. 上下文管理
解決「上下文滿了」的問題：
- 提供腳本自動壓縮上下文
- 抽取最近 10 輪對話的習慣
- 將習慣保存到文件，下次自動載入

### 4. Focus / Hooks（鉤子）
類似 Spring 的生命週期：
- **會話結束時**：自動抽離使用習慣保存
- **下次開始時**：自動載入之前的習慣

---

## 💡 特色功能

| 功能 | 說明 |
|------|------|
| 持續學習 | 越用越好用，自動學習你的習慣 |
| 習慣抽離 | 自動識別你的編碼規範、數據顧習慣 |
| 上下文壓縮 | 不會因為上下文滿了而丟失重要信息 |
| MCP 管理 | 建議不要使用所有 MCP，可選擇性啟用 |
