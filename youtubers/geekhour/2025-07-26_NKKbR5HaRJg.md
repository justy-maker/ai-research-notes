# 15分鐘搭建智能客服

- **頻道**: GeekHour
- **日期**: 2025-07-26
- **連結**: https://www.youtube.com/watch?v=NKKbR5HaRJg
- **時長**: 約 15 分鐘

## 影片概述

本影片介紹如何使用 MaxKB 開源智能體平台快速搭建一個企業內部的智能客服助手。通過結合知識庫和大型語言模型（如 DeepSync），可以實現基於企業內部知識的問答功能。

## 主要技術要點

- **MaxKB**: 開源智能體平台，用於構建智能客服系統
- **Docker**: 容器化平台，用於打包、分發和運行應用程式
- **知識庫**: 儲存企業內部知識的資料庫，智能客服助手可從中檢索相關資訊
- **大型語言模型 (LLM)**: 用於生成回答的 AI 模型，例如 DeepSync
- **RAG (Retrieval-Augmented Generation)**: 從知識庫檢索相關內容後結合 LLM 生成回答

## 實作步驟

### 1. 安裝 MaxKB

**離線安裝方式：**
1. 從 MaxKB 官方網站 (MaxKB.cn) 下載最新版本的離線安裝包
2. 將安裝包上傳到伺服器（可使用 XSHL、SCP 或其他 FTP 工具）
3. 登錄到伺服器（Linux 虛擬機或雲伺服器）
4. 使用 `tar` 命令解壓縮安裝包
5. 進入解壓縮後的目錄
6. 執行 `install.sh` 安裝腳本

**線上安裝方式（Docker）：**
1. 確保系統已安裝並啟動 Docker
2. 複製官方網站提供的 Docker 命令
3. 在終端執行該命令

### 2. 配置 MaxKB

1. 在瀏覽器中輸入伺服器 IP 地址加上端口號 8080（例如 `http://<server_ip>:8080`）
2. 使用預設的管理員帳號和密碼（Admin/MassKBEd123）登錄

### 3. 配置大型語言模型

1. 在管理後台中，點擊「模型 Type-1」
2. 點擊「添加模型」按鈕
3. 選擇模型類型（例如 DeepSync）
4. 填寫模型名稱
5. 填寫 API 密钥
6. 保存配置

### 4. 創建知識庫

1. 在管理後台中，點擊「知識庫」
2. 點擊「創建」按鈕
3. 選擇「通用知識庫」
4. 填寫知識庫名稱和描述
5. 創建知識庫

### 5. 上傳知識庫文檔

1. 點擊「上傳文檔」按鈕
2. 上傳企業相關文檔（公司介紹、產品說明書、服務條款、FAQ 等）
3. 確保文檔內容以問答形式整理，避免包含大量圖片

## 技術細節

### Docker 部署
- MaxKB 使用 Docker 進行部署，會自動拉取相關鏡像
- 使用 `docker images` 查看鏡像信息
- 使用 `docker ps` 或 `docker container ls` 查看容器運行狀態

### 配置文件
- 配置文件包含 Redis 地址、端口和密碼
- PoseGrass 的地址、端口、用戶名和密碼
- MassKB 的服務端口等信息

### RAG 工作原理
1. 用戶提出問題
2. 智能客服從知識庫中檢索相關內容
3. 將內容和問題一起發給大型語言模型
4. LLM 生成最終回答

### 文檔要求
- 最好是以問答形式整理的文本
- 避免包含大量圖片或非文本內容
- 可下載 XL 模板作為參考
