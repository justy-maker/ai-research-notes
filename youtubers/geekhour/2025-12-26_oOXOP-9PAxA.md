## 影片概述

本影片主要介紹 Fish shell，一個使用者友善的命令行 shell。影片涵蓋了 Fish shell 的基本概念、安裝、配置、以及一些進階功能，例如使用 OMF (Oh My Fish) 管理插件和主題，以及命令縮寫功能。此外，影片還說明了如何將 Fish shell 設定為系統預設 shell，以及如何在 VS Code 等程式碼編輯器中配置 Fish shell。

## 主要技術要點

### Fish Shell 簡介
- Friendly Interactive Shell 的縮寫
- 預設提供自動補全、語法高亮等現代化功能
- 相較於 Bash 和 Zsh，對新手更友善，語法更簡潔

### 安裝方式
- 透過包管理器（如 Homebrew、APT、YUM）進行安裝
- 一行命令即可完成下載和安裝

### 配置方式
- 主要配置文件位於 `~/.config/fish/config.fish`
- 可以使用 `fish_config` 命令，透過瀏覽器介面進行配置（在本地 8000 端口開啟）
- 可配置主題、函數、別名等

### 設定為預設 Shell
1. 找到 Fish shell 的路徑（使用 `which fish` 命令）
2. 將 Fish shell 的路徑添加到 `/etc/shells` 文件中（需要管理員權限）
3. 使用 `chsh -s <fish_path>` 命令更改當前使用者的預設 shell

### VS Code 配置
- VS Code 的終端不會自動同步系統預設 shell
- 需要在 VS Code 的 `settings.json` 文件中手動配置 terminal 相關屬性

### OMF (Oh My Fish)
- Fish shell 的插件管理器
- 安裝命令：`curl -L https://get.oh-my.fish | fish`
- 常用命令：
  - `omf list`：列出當前生效的插件和主題
  - `omf theme`：列出所有可用的主題
  - `omf install <plugin_name>`：安裝插件
  - `omf theme <theme_name>`：啟用主題
  - `omf update`：更新框架、主題和插件

### 命令縮寫 (Abbreviations)
- 使用 `abbr -a <abbreviation> <command>` 命令創建縮寫
- 縮寫預設只在當前終端會話中生效
- 若要讓縮寫在所有會話中生效，需要將其添加到配置文件中

## 技術細節

- **配置文件**：`~/.config/fish/config.fish` 是主要配置文件，每次啟動時載入
- **`fish_config` 命令**：在本地 8000 端口啟動瀏覽器配置介面
- **語法差異**：Fish shell 使用 `set` 命令設置變量，語法與 Bash/Zsh 有些不同
