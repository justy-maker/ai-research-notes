# Andrej Karpathy：軟體再次改變

## 影片資訊

## 核心觀點（3-5 個重點）

1.  **軟體發展的三個階段：** Karpathy 提出軟體發展經歷了三個階段：Software 1.0 (傳統程式碼)、Software 2.0 (神經網路權重)、Software 3.0 (大型語言模型，LLM)。
2.  **LLM 的程式設計方式：** Software 3.0 的核心是透過自然語言提示 (Prompt) 來程式設計 LLM，這是一種全新的程式設計範式。
3.  **LLM 的多重角色：** LLM 既像電力等基礎設施，又像晶圓廠，更像作業系統，擁有複雜的生態系統。
4.  **LLM 的普及與應用：** LLM 的普及路徑與傳統技術不同，首先在消費者端普及，然後才被企業和政府採用。
5.  **未來發展方向：** 個人運算革命尚未在 LLM 領域發生，未來可能出現基於 LLM 的個人運算設備和應用。

## 詳細內容摘要（500-800 字）

Andrej Karpathy 在本次演講中探討了人工智慧時代軟體的變革。他認為，軟體正在經歷一次根本性的轉變，而現在正是進入這個行業的絕佳時機。他將軟體發展劃分為三個階段：

*   **Software 1.0：** 這是傳統的程式碼，由開發者直接編寫，用於指示電腦執行特定任務。
*   **Software 2.0：** 這是基於神經網路的權重，開發者不再直接編寫程式碼，而是透過調整資料集和運行優化器來訓練神經網路。Hugging Face 就像是 Software 2.0 領域的 GitHub。
*   **Software 3.0：** 這是指透過大型語言模型 (LLM) 進行程式設計，開發者使用自然語言提示 (Prompt) 來控制 LLM 的行為。這種方式使得程式設計語言變成了英語，極大地降低了程式設計的門檻。

Karpathy 以特斯拉的自動駕駛系統為例，說明了軟體發展的演進。最初，自動駕駛系統大量使用 C++ 程式碼 (Software 1.0)，後來隨著神經網路 (Software 2.0) 的發展，許多原本由 C++ 程式碼實現的功能被遷移到神經網路，使得系統更加高效和智能。現在，LLM (Software 3.0) 的出現，正在進一步改變軟體開發的模式。

他認為，LLM 具有多重角色。首先，它們像電力一樣，具有基礎設施的特性。OpenAI、Google 等公司投入大量資金訓練 LLM，並透過 API 提供服務，使用者按需付費。其次，LLM 又像晶圓廠一樣，需要大量的資本投入和技術積累。最後，Karpathy 認為 LLM 最像作業系統，它們不僅僅是簡單的商品，而是具有複雜的軟體生態系統。LLM 的生態系統與作業系統的生態系統非常相似，既有像 Windows、macOS 這樣的閉源供應商，也有像 Linux 這樣的開源替代方案。

Karpathy 還指出，LLM 的普及路徑與傳統技術不同。通常，新技術首先被政府和企業採用，然後才普及到消費者。但 LLM 卻是反過來的，它們首先在消費者端普及，例如幫助人們煮雞蛋，然後才被企業和政府採用。

他認為，目前 LLM 領域還沒有出現個人運算革命。現在的 LLM 計算仍然非常昂貴，使用者只能透過網路與雲端的 LLM 互動。但他相信，未來可能會出現基於 LLM 的個人運算設備和應用。

## 我的評論（對創業者/開發者的啟發）

Andrej Karpathy 的演講為創業者和開發者提供了寶貴的啟示：

*   **擁抱新的程式設計範式：** Software 3.0 代表著一種全新的程式設計範式，創業者和開發者應該積極學習和掌握這種新的程式設計方式，利用 LLM 的強大能力來開發創新的應用。
*   **關注 LLM 的生態系統：** LLM 的生態系統正在快速發展，創業者和開發者應該密切關注 LLM 的生態系統，尋找新的機會和合作夥伴。
*   **探索 LLM 的應用場景：** LLM 的應用場景非常廣闊，創業者和開發者應該積極探索 LLM 在各個領域的應用，開發出能夠解決實際問題的產品和服務。
*   **思考 LLM 的未來：** LLM 的未來充滿了無限可能，創業者和開發者應該思考 LLM 的未來發展方向，例如基於 LLM 的個人運算設備和應用，並為之做好準備。
*   **不要害怕改變：** 軟體行業一直在不斷變化，創業者和開發者應該擁抱變化，不斷學習和適應新的技術和趨勢，才能在激烈的競爭中脫穎而出。

Karpathy 的演講不僅僅是對軟體發展趨勢的分析，更是一份行動指南，鼓勵創業者和開發者積極參與到這場變革中，共同創造軟體行業的未來。
