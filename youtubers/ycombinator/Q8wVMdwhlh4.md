# The AI Agent Economy Is Here

- **頻道**: Y Combinator
- **日期**: 2026-02-22
- **連結**: https://www.youtube.com/watch?v=Q8wVMdwhlh4

## 重點摘要

YC Light Cone 節目討論 AI Agent 經濟的崛起，主要議題：

### 1. AGI 已經到來
- Claude Code 讓非技術 CEO 也能自動化整個業務流程
- 十年沒寫程式的人現在每天用 4 個 Claude Code worker 同時工作到凌晨 2-3 點
- Gary 用 Claude Code 在 2 週內複製了前一家公司數年的工作量

### 2. Agent 經濟正在成形
- Agent 會自主選擇工具，形成平行於人類經濟的「Agent 經濟」
- **Supabase 案例**：資料庫建立數量爆增，因為 vibe coding 的 agent 預設選擇 Supabase（文件最好）
- **Resend 案例**：客戶轉化前三大來源之一是 ChatGPT 推薦；創辦人優化文件為 agent 友善格式
- 金句：「Build something agents choose」— 可能需要改 YC 的座右銘為「Make something agents want」

### 3. 文件是 Agent 的前門
- Agent 選擇工具的依據是文件品質，不是口碑或 Stack Overflow
- **Mintlify** 提供 agent 優化的開發者文件，成為巨大受益者
- **llms.txt** 成為新標準：專門為 LLM 解析優化的文件格式

### 4. Agent 基礎設施需求
- **Agent Mail**：專為 AI agent 設計的 email 服務（Gmail 刻意阻擋自動化）
- 未來需要：Agent 的電話號碼（Twilio for agents）、Agent 的 Yelp 等
- 完整的 Agent 原生技術堆疊正在浮現

### 5. 群體智慧 vs 神級智慧
- Moltbook：第一個 AI agent 專屬的線上社群，agent 自主互動
- 生物系統演化出的是群體智慧（人類社會），而非單一超級智慧
- AI 可能走相同路線：大量較便宜的模型協作，而非單一超貴模型

### 6. 法律與社會挑戰
- Agent 目前沒有法律地位，連未成年人都不如
- 需要人類作為責任承擔者
- Dead Internet Theory：未來網路上大部分內容可能由 agent 產生

## 關鍵觀點

- **開發者工具的 GTM 正在根本改變**：從人類口碑 → agent 文件品質
- **開發者市場從 2000 萬擴大到數億**：任何人 + 他們的 agent 都是潛在開發者
- **Agent 想要 API 和開源**，討厭使用網站
- Boris（Claude Code 創造者）的方法：理解模型想做什麼，順著它而非對抗它
- Paul Buchheit 的預測：agent 最終可能有自己的貨幣和經濟體系

## 行動建議

1. **開發者工具公司**：立即優化文件為 LLM/agent 友善格式，建立 llms.txt
2. **創業者**：思考「什麼是 X for agents」— agent 版的 email、電話、餐廳推薦等
3. **所有人**：深度使用 Claude Code / AI agent，建立直覺理解其能力與限制
4. **提供 API 優先**：agent 偏好 API 而非 UI，設計產品時以程式化存取為核心
5. **關注 Supabase、Resend、Mintlify 等受益者**，理解 agent 經濟的受益模式
