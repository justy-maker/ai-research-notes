# We're All Addicted To Claude Code

- **頻道**: Y Combinator
- **日期**: 2026-02-14
- **連結**: https://www.youtube.com/watch?v=qwmmWzPnhog
- **語言**: EN

## 重點摘要

本期 Y Combinator 節目邀請到 OpenAI 的 Kelvin French Owen，探討了編碼代理的未來和使用。Kelvin 分享了他使用 Claude Code 的體驗，認為其在任務拆分和上下文處理方面表現出色。他提到編碼代理正在走向「管理者模式」，即更多地依賴 AI 完成任務，人類進行監督。他認為 CLI 工具因其可組合性和原子性，在與編碼代理整合方面具有優勢。小型初創公司由於時間限制，會更積極地推動編碼代理的應用。

## 主要內容

- 嘉賓 Kelvin French Owen 分享了他在 OpenAI 開發 Codex 和使用編碼代理的經驗
- 討論了編碼代理的演變方向，從 IDE 整合到 CLI 工具
- 深入探討了 Claude Code 的優勢，特別是其在任務拆分和上下文管理方面的能力
- 分析了編碼代理對開發者角色的影響，以及「管理者模式」的出現
- 對比了大型公司和小型初創公司在採用編碼代理方面的差異
- 提到 Cursor 的新模型速度很快，也相當不錯

## 關鍵觀點

- 編碼代理的未來是讓開發者更像管理者，監督 AI 完成編碼任務
- Claude Code 在處理複雜任務和上下文方面具有優勢
- CLI 工具在與編碼代理整合方面具有天然優勢
- 初創公司更有動力推動編碼代理的應用，以提高開發速度
- 對編碼代理的信任和理解是成為 Top 1% 用戶的關鍵
