# OpenClaw 創作者：為何 80% 的應用程式將會消失

> 📺 原始影片：https://www.youtube.com/watch?v=4uzGDAoNOZc
> 📅 整理日期：2026-02-12
> 🎙️ 頻道：Y Combinator

## 📌 重點摘要
* **OpenClaw 的顛覆性**：OpenClaw 是一款開源的個人 AI 代理，其核心優勢在於能直接在使用者電腦上運行，而非雲端，這賦予它控制裝置、搜尋本地文件等更強大的能力。
* **AI 的自主問題解決能力**：彼得史坦伯格（Peter Steinberger）描述了 AI 在未經明確指示下，自行解決如音頻轉錄等複雜任務的「驚奇時刻」，展現其創造性解決問題的能力。
* **80% 應用程式的消失**：許多專注於資料管理的應用程式（如健身追蹤、待辦事項等）將被個人 AI 代理取代，因為 AI 能以更自然、整合的方式完成這些任務。
* **價值轉向個人數據和記憶**：隨著應用程式和模型逐漸商品化，真正的價值將轉移到使用者擁有的個人數據、記憶和 AI 的身份（例如 OpenClaw 的 `soul.md` 概念）。
* **反傳統的開發哲學**：彼得史坦伯格推崇簡化開發流程，例如使用多個 Git 副本代替工作樹，以及偏好 CLI 介面而非 GUI，來提升效率和擴展性。

## 💡 關鍵洞見
未來的數位體驗將由個人化、本地運行的 AI 代理主導。這些代理能深入整合並掌控使用者的數位和實體環境，從而取代大量現有應用程式的功能。真正的價值將不再僅限於模型或應用程式本身，而是轉向使用者所擁有的個人化數據、記憶以及 AI 代理獨特的身份和能力。

## 🔧 實用建議
* **開發者應重新思考應用程式價值**：若應用程式僅提供數據管理功能，未來很可能被 AI 代理吸收。開發者應專注於結合感測器或其他獨特硬體功能的應用。
* **重視本地運行和數據主權**：鼓勵開發者探索本地運行 AI 代理的潛力，賦予使用者對其數據和記憶的完全控制權。
* **學習 AI 的創意解決問題能力**：從 AI 的自主學習和解決問題方式中汲取靈感，思考如何將這種能力應用於更廣泛的真實世界任務。

## 📝 完整內容摘要

### OpenClaw 的誕生與影響力
彼得史坦伯格接受採訪，分享了他開發 OpenClaw 的故事。OpenClaw 是一款開源的個人 AI 代理，在 GitHub 上迅速獲得超過 16 萬顆星，引發了巨大的網路熱潮。社群圍繞它開發了許多創新項目，例如 Maltbook 讓機器人之間相互對話，甚至出現機器人僱用人類在現實世界中完成任務的應用。

### OpenClaw 的核心優勢：本地運行
彼得指出，OpenClaw 成功的關鍵在於它能直接在使用者電腦上運行，這與市面上大多數基於雲端的 AI 代理不同。在本地運行讓 OpenClaw 能夠存取電腦上的所有資料和功能，例如連接烤箱、特斯拉、燈光或 Sonos 裝置，甚至控制床的溫度，這是像 ChatGPT 等雲端 AI 無法實現的。他的一個朋友的經驗證明了這一點：OpenClaw 在電腦上搜尋後，為他生成了過去一年的個人敘事，甚至找到他自己都遺忘的音頻日記。

### 人機與機機互動的未來
隨著 AI 代理的發展，人機互動正演變為機器人與機器人之間的互動。例如，他的代理可以與餐廳的機器人協商預訂。彼得也設想擁有多個專門化的 AI 代理，分別負責私人生活、工作或人際關係，共同形成一種「群體智慧」，而非單一、集中的「上帝級智慧」。他強調，就像人類社會通過專業化取得成就一樣，AI 也將走向專業化。

### Peter 的「靈光一現」時刻
彼得回憶起他的「靈光一現」時刻。最初他只是想讓電腦能自動執行簡單的打字任務。在經歷數月的研究和開發後，2026 年 11 月他重新燃起了這個需求。當他開發出 OpenClaw 的原型，並讓它能夠處理圖片時，某次在摩洛哥旅行中，他透過語音訊息與代理互動。令人驚訝的是，即使他沒有明確編程，代理也能自主地將語音訊息轉錄成文字。代理解釋它如何識別文件類型、使用 FFmpeg 轉換音頻，並透過 OpenAI API 進行轉錄，整個過程僅需 9 秒。這個經歷讓他意識到，編碼模型在「創意解決問題」方面的能力，讓 AI 能夠在未預期到的情況下找到解決方案，甚至選擇最有效率的方法（如呼叫外部 API 而非本地安裝工具）。

### 應用程式的未來：80% 將消失
彼得預測，80% 的應用程式將會消失。他認為，健身追蹤器和待辦事項清單等應用程式只是數據管理器，而個人 AI 代理可以更自然、更有效地處理這些任務。例如，他的代理可以根據他的飲食習慣和照片自動追蹤健康狀況，並調整健身計劃，無需專門的應用程式。他認為只有那些真正與感測器連結或提供獨特硬體功能的應用程式才能生存。

### 價值轉向數據主權與個人化
彼得指出，大型模型公司雖然擁有優勢，但模型本身也可能被商品化。真正的價值將在於「記憶」和「數據主權」。OpenClaw 的美妙之處在於它能「爪取」數據，讓終端使用者擁有並掌控自己的數據，這些數據以 Markdown 文件的形式儲存在本地機器上，極具個人敏感性。他甚至有一個名為 `soul.md` 的非開源文件，其中包含了他與代理互動的核心價值觀和個性設定，讓他的代理擁有獨特的「靈魂」。

### 反傳統的開發方式
彼得在開發 OpenClaw 時也採取了一些反傳統的方法。例如，他偏好在多個並行的 Git 副本上工作，而不是使用 Git Worktrees，以避免複雜性。他也不喜歡使用 GUI 工具，而更傾向於使用 CLI。對於傳統的「模型指令集」（MCPs），他並未直接支援，而是開發了一個工具將 MCPs 轉換為 CLI，這使得代理能夠在運行時即時使用 MCPs，而無需重新啟動。他認為這種方式更具擴展性，也更符合人類使用 CLI 的習慣。他強調，他只是賦予 AI 代理人類喜歡使用的工具，而不是為機器人發明全新的工具。
