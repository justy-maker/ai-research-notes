# 如何最大化利用 Vibe Coding（中文翻譯）

## 影片資訊
- 原標題：How To Get The Most Out Of Vibe Coding
- 連結：https://www.youtube.com/watch?v=BJjsfNO5JTo
- 頻道：Y Combinator

## 核心觀點（3-5 個重點）

1. **制定詳細計畫並逐步實施：** 不要試圖一次性完成整個專案，而是與 LLM 合作制定詳細的計畫，將其分解為小的、可管理的章節，並逐步實施，每次完成一個章節後進行測試和提交。
2. **善用版本控制：** 像使用 Git 一樣，嚴格使用版本控制，以便在 AI 出現問題時可以輕鬆地恢復到已知的工作版本。
3. **編寫高階測試：** 編寫高階整合測試，模擬使用者操作，確保功能端到端正常工作，而不是僅僅測試單元函數。
4. **將 LLM 用於非編碼任務：** 除了編碼之外，還可以利用 LLM 進行 DevOps、設計等任務，以加速開發進程。
5. **重置並重新開始：** 如果 LLM 在修復錯誤時遇到困難，不要多次嘗試，而是重置程式碼庫並重新開始，避免累積多餘的程式碼層。

## 詳細內容摘要（500-800 字）

這段影片分享了如何最大化利用 Vibe Coding 的技巧。Vibe Coding 是一種利用 AI 工具（如 LLM）進行軟體開發的方法，類似於早期的提示工程。影片首先採訪了幾位 YC 春季班的創始人，他們分享了各自在使用 AI 工具進行編碼時的經驗。

其中一位創始人建議，如果 AI 在 IDE 中遇到無法實現或除錯的問題，可以直接將程式碼複製到 LLM 的網站 UI 中，有時可以得到 IDE 無法提供的解決方案。另一位創始人則建議同時使用 Cursor 和 Windsurf 兩個工具，Cursor 速度較快，適合前端開發，而 Windsurf 思考時間較長，適合後端開發。他還會讓兩個工具同時處理相同的任務，然後選擇他更喜歡的結果。還有一位創始人將 AI 視為一種新的程式語言，因此需要提供非常詳細的上下文資訊才能獲得良好的結果。他通常從編寫測試案例開始，確保 LLM 生成的程式碼能夠通過這些測試。

影片主持人 Tom 接著分享了他自己的 Vibe Coding 經驗和建議。他建議初學者可以使用 Replet 或 Lovable 等工具，它們提供易於使用的視覺介面。對於有編碼經驗的人，可以直接使用 Windsurf、Cursor 或 Claude Code 等工具。

Tom 強調，在使用 AI 工具之前，應該先與 LLM 合作編寫一份全面的計畫，並將其保存在專案資料夾中。然後，逐步按照計畫實施，每次完成一個章節後進行測試和提交。他還建議嚴格使用版本控制，以便在 AI 出現問題時可以輕鬆地恢復到已知的工作版本。

此外，Tom 還建議編寫高階整合測試，模擬使用者操作，確保功能端到端正常工作。他指出，LLM 有時會對不相關的程式碼進行不必要的修改，因此測試可以幫助及早發現這些問題。

Tom 還強調，LLM 不僅僅可以用於編碼，還可以利用它們進行 DevOps、設計等任務。例如，他使用 Claude Sonet 3.7 配置 DNS 伺服器，並使用 ChatGPT 創建網站的 favicon。

在修復錯誤方面，Tom 建議首先將錯誤訊息複製到 LLM 中，通常這就足以讓 AI 識別並修復問題。如果錯誤比較複雜，可以要求 LLM 先思考幾種可能的成因，然後再編寫程式碼。每次修復失敗後，都應該重置程式碼庫並重新開始，避免累積多餘的程式碼層。

Tom 還建議編寫 LLM 的指令，並將其保存在 Cursor rules、Windsurf rules 或 Claude markdown 檔案中。他指出，網路上有很多關於如何編寫這些指令的建議。

對於複雜的功能，Tom 建議先在一個乾淨的程式碼庫中建立一個小型的參考實現，然後讓 LLM 參考這個實現，在更大的程式碼庫中重新實現它。

最後，Tom 建議選擇合適的技術堆疊。他發現 AI 在編寫 Ruby on Rails 程式碼時表現出色，因為 Rails 是一個有著 20 年歷史的框架，擁有大量的訓練資料。

總之，這段影片提供了許多關於如何最大化利用 Vibe Coding 的實用技巧，包括制定詳細計畫、善用版本控制、編寫高階測試、將 LLM 用於非編碼任務，以及重置並重新開始。
