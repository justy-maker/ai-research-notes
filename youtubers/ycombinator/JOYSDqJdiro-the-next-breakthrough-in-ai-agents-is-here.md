# AI Agents 的下一個突破就在這裡

## 影片資訊
- 原標題：The Next Breakthrough In AI Agents Is Here
- 連結：https://www.youtube.com/watch?v=JOYSDqJdiro
- 頻道：Y Combinator

## 核心觀點
1.  **Manis 的出現：** 一家名為 Manis 的中國新創公司推出了一款通用型 AI Agent 平台，引起了廣泛關注，被譽為中國的下一個 DeepSeek 時刻。
2.  **多 Agent 架構：** Manis 採用多 Agent 架構，類似於一位主管協調多個子 Agent 完成任務，而非依賴單一大型神經網路。
3.  **功能與應用：** Manis 擅長處理各種實際任務，例如旅行規劃、財務分析、教育內容生成等，並在 Gaia 基準測試中取得了優異成績。
4.  **Rapper 的爭議：** Manis 被一些人視為「Rapper」，即基於現有基礎模型和工具的整合應用，但影片認為成功的 AI 產品往往也屬於此類。
5.  **差異化競爭：** 對於 AI 新創公司而言，關鍵不在於重新發明輪子，而在於如何將現有模型整合為使用者真正喜歡的產品，並建立可持續的差異化優勢。

## 詳細內容摘要
影片介紹了一款名為 Manis 的新型 AI Agent 平台，該平台由一家中國新創公司開發，並迅速引起了業界的關注。與傳統的專用聊天機器人不同，Manis 旨在成為一個真正的通用型 AI Agent，能夠處理各種複雜任務。

Manis 的核心創新在於其多 Agent 架構。它不像傳統的 AI 模型那樣依賴單一的大型神經網路，而是採用類似於企業主管協調團隊的方式。當使用者輸入提示詞後，Manis 會首先通過一個規劃 Agent 將任務分解為多個可管理的子任務，然後將這些子任務分配給不同的子 Agent。這些子 Agent 就像 Manis 內部的專家，每個 Agent 都有自己的專業領域，例如知識、記憶或執行。Manis 可以調用 29 種不同的整合工具，包括自動化網頁導航、安全程式碼執行和文件資訊提取等。最後，執行 Agent 會將所有子任務的輸出整合為最終結果呈現給使用者。

Manis 的底層技術包括動態任務分解演算法和鏈式思考注入技術，前者可以將複雜指令分解為清晰的執行路徑，後者則可以讓 Agent 主動反思和更新計畫，確保穩定性。Manis 使用 Anthropic 的 Claude 3.7 Sonnet 模型，並與 YC 公司的 BrowserUse 和 Startup E2B 的安全雲端沙箱環境等開源工具無縫整合，實現了跨平台執行能力。

Manis 在各種實際應用中表現出色，例如創建旅行行程、進行詳細的財務分析、生成教育內容等。為了衡量 Manis 的能力，影片提到了 Gaia 基準測試，該測試旨在挑戰 AI Agent 的推理、多模態處理、網頁瀏覽和工具熟練程度。Manis 在 Gaia 測試中取得了 86.5% 的分數，僅比人類平均水平低幾個百分點，超越了 OpenAI 的 Deep Research。

儘管 Manis 的基準測試表現令人印象深刻，但它也引發了關於 AI 新創公司在應用層面（即「Rapper」）的本質的討論。有些人認為 Manis 只是一個 Rapper，因為它只是將現有的基礎模型和工具整合在一起。然而，影片認為，許多成功的 AI 產品實際上都屬於 Rapper 類別，例如 Cursor、Windsurf 和 Harvey。這些產品通過整合現有的 LLM、API 和特定領域的工具，為使用者提供了有價值的服務。

影片強調，對於 AI 新創公司而言，關鍵不在於重新發明輪子，而在於如何將現有模型整合為使用者真正喜歡的產品，並建立可持續的差異化優勢。這可能包括投資於難以複製的專有評估、將工作流程深度嵌入到特定使用者的習慣中，或者與競爭對手難以訪問的平台或數據集進行整合。

## 我的評論
這部影片深入分析了 Manis 這個新型 AI Agent 平台，並引發了關於 AI 應用開發模式的思考。對於創業者和開發者來說，有以下幾點啟發：

*   **擁抱多 Agent 架構：** Manis 的成功證明了多 Agent 架構在處理複雜任務方面的優勢。開發者可以考慮採用類似的架構，將任務分解為更小的、可管理的子任務，並分配給不同的 Agent 處理。
*   **不要害怕成為 Rapper：** 影片指出，許多成功的 AI 產品實際上都屬於 Rapper 類別。創業者不必過於追求底層技術的突破，而應該專注於如何將現有模型和工具整合為使用者真正需要的產品。
*   **建立可持續的差異化優勢：** 在競爭激烈的 AI 市場中，建立可持續的差異化優勢至關重要。創業者可以通過投資於專有評估、深度嵌入使用者工作流程或與獨特的數據集進行整合等方式來實現這一目標。
*   **使用者體驗至上：** 無論底層技術如何，最終決定產品成敗的還是使用者體驗。創業者應該始終將使用者需求放在首位，並不斷優化產品的易用性和功能性。

總之，Manis 的出現為 AI Agent 的發展帶來了新的可能性。創業者和開發者應該從中汲取靈感，並積極探索 AI 在各個領域的應用。
