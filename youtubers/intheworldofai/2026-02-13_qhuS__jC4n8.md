### 📝 詳細內容摘要

**問題：Claude 缺乏持久記憶**

*   Entropic 的 Claude 模型的主要限制之一是缺乏真正的跨會話持久記憶。
*   由於會話的無狀態設計以及相較於 Gemini 等模型較小的上下文窗口，每次會話結束後，模型都會重新開始。
*   這導致 Claude 忘記專案上下文、決策和過去的工作，需要重複解釋所有內容，浪費時間和 token。
*   這種重複的「重新啟動」會消耗 token 預算，減少用於實際推理和高品質輸出的 token。

**解決方案：Claude Mem**

*   Claude Mem 將 Cloud Code 變成了一個可以跨會話記住專案歷史的工具。
*   它自動捕獲 Claude 的工具使用、決策和觀察，並將其壓縮存儲到帶有向量搜尋的本地資料庫中。
*   它可以將相關上下文注入到未來的會話中，使 Claude 真正記住您的專案。
*   它還允許您通過 memsearch 和 mcp 工具使用自然語言搜尋密碼。
*   它是開源的，安裝後會自動在後台運行。

**Claude Mem 的效果**

*   影片使用相同的詳細儀表板提示進行了兩次測試，一次在無狀態的 Claude 會話中進行，一次啟用了 Claude Mem。
